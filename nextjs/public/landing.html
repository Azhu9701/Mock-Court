Usage Guide

六合一的思辨引擎

六种附体模式，从快速咨询到多魂合议，覆盖每一个思辨场景。

1 单魂 快速咨询

单一魂独立输出，适合快速提问与直接获取某个思想家的观点。输入问题，选择一个魂，即刻得到带有该角色独特思维方式的回答。

2 合议 核心模式

多魂并行流式输出 + 实时语义碰撞检测 + 三级门控干预 + 辩证综合。每个魂作为独立 tokio task 推进，碰撞检出后立即注入干预信号，最终由辩证综合官整合为综合报告。

3 辩论 对立视角

两列对立布局，正方与反方各自推进，中间裁决栏实时呈现裁判判定。适合需要深度对比两种对立观点的场景。

4 接力 阶段递进

横向时间轴，每个阶段由不同魂接力推进。前一个魂的输出作为下一个魂的输入，形成递进式的思考链条。

5 学习 费曼学习法

魂间教学模式——一个魂作为老师分析学习者的盲区，出题考察，评分反馈。用费曼学习法循环提升理解深度。

6 实践开口 方法论实践

方法论实践模式——将思辨转化为行动。每个会话结束后三问闭环：你会做什么？谁该在场？什么判断错了？

Configuration

五分钟上手

从安装到第一次召唤，只需要三步。

1

安装 API Keys

将 API Key 写入 data/apikeys.json（此文件已在 .gitignore 中）：

{
  "deepseek": "sk-your-deepseek-key",
  "openai": "sk-your-openai-key",
  "claude": "sk-your-claude-key"
}

至少需要一个 AI 提供商的 API Key 即可启动。

2

可选：编辑配置文件

编辑 config/default.yaml 自定义路径和端口：

data_dir: "./data"
souls_dir: "./data/souls"
server_port: "3096"
nextjs_port: "3000"
3

一键启动

macOS / Linux 用户使用 bash start.sh 一键启动 API + 前端。Windows 用户从 MSI 安装包启动即可。

# 一键启动
bash start.sh

# 或分别启动
cargo run -p api --release
cd nextjs && pnpm dev

Advanced

不只是聊天界面

每一个设计决策都在回答同一个问题——怎么让你真正思考，而不是消费思考。

语义碰撞检测

三路径并行：L1 关键词规则(微秒) → L2 trigram Jaccard 相似度(毫秒) → L3 Flash LLM 判定(秒)。

L1 关键词规则 → 微秒级
L2 Trigram Jaccard → 毫秒级
L3 Flash LLM → 秒级
碰撞类型: 矛盾 / 差异 / 前提分歧 / 补充挑战 / 冗余 / 盲区互补

魂记忆图谱

基于 petgraph 的有向图：前提 → 结论的推导链、跨魂融合边。BFS 矛盾检测 (O(n)，零 LLM 成本)。

前提 → 结论 → 新前提
Integrates → 跨魂融合边
BFS 矛盾检测 · DFS 前提动摇传播

蛇皮指数

(无效 + 部分有效 × 0.5) / 总会话 × 100 — 数字越高，说明你消费的思辨越多、行动越少。

≥70 厚重   40-69 中等
15-39 较薄   <15 接近蜕皮

@mention · 搜索 · 成本追踪

输入框 @魂名 即时召唤。SearXNG 联网搜索注入上下文。Token 消耗实时追踪，模型智能路由自动选择最优模型。

@马克思 → 直接召唤回复
SearXNG → 自托管元搜索
Model Router → 自动模型选择

审查官权威机制

算法匹配魂 → 审查官一审(pass/reject + 缺失视角) → 补充匹配 → verified_souls 作为唯一权威。讨论启动前审查官先行提问，答案注入魂共用的时代背景。

算法匹配 → 审查官一审 → 终裁
verified_souls 是唯一权威，算法匹配只作参考

自适应合议拓扑

5 种拓扑自动选择：Minimal / ClusteredParallel / FullMesh / Oppositional / SequentialLadder。基于魂多样性 + 任务复杂度 + 预算约束的决策树。30 秒无碰撞自动降级，节省 LLM 成本。

Minimal → FullMesh → Oppositional
30s 无碰撞 → 自动降级节省成本
