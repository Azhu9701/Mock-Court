# Business Rules — F3: Possession UI

## 1. Wizard 步骤规则 (Q2: B)

| 规则 | 描述 |
|------|------|
| BR1.1 | 5 步线性流程，不可跳步，可返回前一步 |
| BR1.2 | Step 1 (模式): 6 类模式卡片，单击选中 |
| BR1.3 | Step 2 (选魂): 从 `GET /souls` 获取列表，搜索过滤 |
| BR1.4 | 单魂模式限制选 1 魂，辩论限制选 2 魂，合议限制 2-10 魂 |
| BR1.5 | 超出限制时禁用多选并提示 |
| BR1.6 | Step 3 (任务): textarea，非空校验 |
| BR1.7 | 辩论模式额外显示辩题输入框 |
| BR1.8 | Step 4 (确认): 显示所有配置，确认后调用 POST /possess |
| BR1.9 | `?preset=mode&souls=a,b` URL 参数可跳过 Wizard 直达 Step 4 |

## 2. WebSocket 流式规则

| 规则 | 描述 |
|------|------|
| BR2.1 | WS 连接：`ws://127.0.0.1:3096/ws/possess/{sessionId}/main` |
| BR2.2 | 连接失败时显示错误 + 重试按钮 |
| BR2.3 | SoulChunk: 追加内容到对应魂的聊天气泡，滚动到底部 |
| BR2.4 | SoulDone: 标记魂完成，气泡变灰 |
| BR2.5 | SoulError: 显示红色错误徽章 |
| BR2.6 | SessionComplete: 所有魂完成，显示总结 |
| BR2.7 | 断线自动重连 (exponential backoff, max 3 retries) |

## 3. 聊天显示规则 (Q1: B)

| 规则 | 描述 |
|------|------|
| BR3.1 | 魂的消息：左侧气泡 + 魂名 + 头像 (首字母) |
| BR3.2 | 用户的追问：右侧气泡（后续 F6 实现） |
| BR3.3 | 流式内容追加时气泡高度自动增长 |
| BR3.4 | 新 chunk 到达时平滑滚动到底部 |
| BR3.5 | Sysnthesis 消息：居中系统气泡，不同底色 |

## 4. 主副面板规则 (Q3: B)

| 规则 | 描述 |
|------|------|
| BR4.1 | 左侧概览面板：魂列表 (名称 + 状态图标)，宽度 240px |
| BR4.2 | 右侧主面板：聚焦魂的完整聊天输出 |
| BR4.3 | 单击概览中的魂名切换到该魂的主视图 |
| BR4.4 | 有魂完成时自动切换到下一个未完成的魂 |
| BR4.5 | 所有魂完成后自动显示 Synthesis 面板 |
| BR4.6 | 概览状态图标：○等待 ●进行中 ✓完成 ✗错误 |

## 5. 上下分屏规则 (Q4: B)

| 规则 | 描述 |
|------|------|
| BR5.1 | 上半屏为 Soul A，下半屏为 Soul B |
| BR5.2 | 分隔线可拖拽调整比例 (默认 50:50) |
| BR5.3 | 两魂同时开始流式输出（并行） |
| BR5.4 | 裁决结果在底部显示 |

## 6. 错误处理

| 规则 | 描述 |
|------|------|
| BR6.1 | API 调用失败 → toast error |
| BR6.2 | WS 连接失败 → 显示 "连接已断开，正在重连..." |
| BR6.3 | 重连 3 次失败 → 显示 "无法连接，请稍后重试" + 手动重试按钮 |
| BR6.4 | 刷新页面 → 通过 sessionId 重新连接 WS (如仍活跃) |
