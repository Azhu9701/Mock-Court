# Snake Skin Framework — 框架全景

## 一句话

一个前后端组合，让你能用 **`snake init`** 一条命令生成多 AI Agent 并行推理 Web 应用。换套配置就能切换业务领域。

## 能做什么

| 场景 | 怎么用 |
|------|--------|
| **法律顾问系统** | 3 个 Agent（合同专家、诉讼律师、法规研究员）并行分析同一案件，碰撞后给出综合意见 |
| **医疗会诊** | 多个专科 AI 同时诊断，交叉验证，标记矛盾结论 |
| **金融分析** | 基本面/技术面/政策面 Agent 并行输出，自动合成投资建议 |
| **代码审查** | 安全 Agent + 性能 Agent + 架构 Agent 同时审查，碰撞后给出优先级排序 |
| **客服路由** | 按问题类型自动匹配最佳 Agent 组合，流式输出解决方案 |

## 核心链路

```
用户输入 → 任务分析 → Agent 匹配 → 并行调用 AI → 流式推送前端 → 碰撞检测 → 辩证综合
```

每一步都可以自定义或替换。

## 三件事就能切领域

1. **写一个 `domain.yaml`** — 定义你的术语（Agent 叫什么、用户叫什么）、坐标维度、综合 Prompt 模板
2. **写 Agent 定义文件** — Markdown 格式，描述每个 Agent 的人设、专长、Prompt
3. **（可选）写自定义模式** — 如果 6 种内置模式不够用，注册你自己的推理流程

不改任何 Rust/TypeScript 代码。

## 架构分层

```
┌────────────────────────────────────┐
│ Agent 人格层                        │  ← 你主要写这一层
│ Agent 定义文件 (.md)、Prompt、人设  │
├────────────────────────────────────┤
│ 领域模板层                          │
│ domain.yaml 驱动术语/维度/模板      │  ← 切领域换这一层
├────────────────────────────────────┤
│ 通用编排内核                        │
│ Gateway → Engine → Stream → WS     │  ← 框架提供，不改
│ + Tool 系统 + Storage + Memory     │
└────────────────────────────────────┘
```

## 技术栈

| 层 | 技术 |
|----|------|
| 后端框架 | Rust + Axum 0.7 + Tokio |
| AI 网关 | Claude / OpenAI / DeepSeek / LM Studio 统一适配 |
| 实时通信 | WebSocket + SSE |
| 存储 | SQLite (WAL) + FTS5 全文搜索 |
| 内存索引 | DashMap 并发哈希 |
| 记忆图谱 | petgraph 图结构 |
| 前端 | Next.js 16 + React 19 + shadcn/ui + Tailwind CSS |
| 脚手架 | Rust CLI (clap) |
