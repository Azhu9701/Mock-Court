# 魂目录 (souls/)

这是公开的"魂"目录。当前发布版本仅包含两个**演示用魂**：

- `海绵宝宝.md` — 比奇堡乐观主义代表，作为生活哲学示例
- `马斯克.md` — 跨领域技术先知，作为产业资本逻辑示例

完整的 28+ 魂（包括马克思、列宁、毛泽东、未明子、孔子、庄子、尼采、黑格尔、鲁迅、葛兰西、法农、波伏娃、拉康、伊本赫勒敦、费曼、Karpathy、Aaron Swartz 等）**不在本仓库中发布**。

## 为什么这样安排

Snake Skin的设计目标是"实践与理论的反馈循环"——它**不是**一个"拿来就能跑"的独立工具。完整的魂体系需要：

1. **集体炼化的隐性知识**——每个魂的召唤词、ismism 坐标、审查规则、互补/互斥关系，都是项目集体通过线下行动 + 反复试错沉淀下来的，不是文本本身能传达的。
2. **审查官机制**——魂之间的合议、辩论、综合，需要审查官（如未明子）在场，否则容易退化为意识形态合奏团。
3. **拓扑规划**——多魂场景下的角色分配（议长、批判、综合）需要拓扑配置，这本身是行动经验。

把这些以"开源"的形式直接发布，相当于把劳动成果交给资本平台无成本征用，违背项目宗旨（详见 `LICENSE` 附加条款）。

## 如何获取完整魂目录

完整魂目录在**内部 repo**（自托管 Forgejo）维护。访问权通过下列任一路径获得：

- 参加项目相关的线下行动或工作坊
- 经两位现有内部成员担保
- 通过审核流程（递交项目使用计划，说明非剥削性集体用途）

具体联系方式将随项目正式发布在 README 中给出。

## 如何启用完整魂目录（已获取后）

将内部 repo 克隆到本地任意位置，然后通过环境变量指向：

```bash
export WANMINFAN_SOULS_INTERNAL_DIR=/path/to/internal-souls
./start.sh
```

或将内部 repo 软链到 `data/souls-internal/`：

```bash
ln -s /path/to/internal-souls data/souls-internal
./start.sh
```

代码会自动同时加载 `data/souls/`（公开演示）和 `data/souls-internal/`（完整集合），同名魂以内部版本为准。

## 自己写魂

完全鼓励自己写魂放在 `data/souls/` 里。每个魂是一个 markdown 文件，包含 YAML frontmatter（元数据 + 召唤关键词）和 markdown 正文（系统提示词）。可参考 `海绵宝宝.md` 和 `马斯克.md` 的结构。
