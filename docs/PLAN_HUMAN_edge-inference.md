# PLAN for Human — 端侧辩证推理·人的行动路线

> AI 能跑量化管线、写推理引擎、搭 P2P 网络。以下七件事，AI 做不了。这是你的 PLAN。

---

## 1. 判断方向

AI 给选项，不给判断。

**你一直在做的事：**

- 2500 字目标该不该设？→ 试了三轮，发现 3B MoE 崩溃、27B 密集不用设也到。**你判断：撤回。**
- "每段不超过 3 句"该不该去？→ AI 不确定。**你判断：去掉。** 结果是 35-83% 的篇幅提升。
- L4 差云端一档，加显式指令还是接受天花板？→ **你判断：加两步指令。** 结果 27B 追上云端。

AI 跑测试出数据，但"跑哪个测试""数据说明什么"是你的判断。这不可委托。

**接下来需要判断的事：**

- 什么时候从 prompt 优化转到代码工程？当前 27B 已接近天花板，继续抠 prompt 的边际收益在下降
- Core ML 还是 mlc-llm 作为 iOS 主线？Apple 可能锁 Core ML 生态，mlc-llm 跨平台但性能差一档
- 要不要提前上 WASM 逃生通道？浏览器推理比原生慢 3-5 倍，但不受 App Store 审查

---

## 2. 坚持到底

AI 没有 stake。项目没人用的时候它不会留下来。

你的实践现场已经跑出来了——4-5 人车间、20 人合作社、滴滴的骑手、本地部署的开发者。这些不是 AI 生成的用例。是你自己在场、自己记录的。

**坚持意味着：**

- 技术路线图 5 年。中间会有人离开、社区会冷、工具链会断。留下来的人做决定
- 魂的实践记录（Marx 的 practice_observations）是持续积累的——不是一次性工程。每个季度至少跑一次滴滴同题评测
- 当开源模型许可证收紧（Meta 已经在动 Llama）、监管收紧（欧盟 AI Act 后续条款）时，有人决定"继续用还是切方案"

---

## 3. 建立信任

一个骑手把派单数据交给你，不取决于你的加密方案——取决于他信不信任你这个人。

AI 不建立关系。AI 不一起吃过亏。

**具体要做的：**

- 找到第一批使用者。不是发帖等他们来——是去他们在的地方。微信群、骑手休息点、修车铺
- 不推销。"这个 App 能帮你分析平台规则"——不如当场拿他的一个问题跑一遍，让他看结果
- 不承诺 AI 不能兑现的东西。不说"这个能帮你打赢官司"。说"这个能帮你看清你在什么位置"
- 数据主权从第一天就交出去。使用者的数据存本地、不经过任何服务器。技术上做到，口头上说清楚

---

## 4. 说"不"

路线图里"不做什么"那一段——不训基座、不造引擎、不做通用助手——每个否定都决定了资源往哪投。

AI 的默认是"都可以试试"。说"不"需要人。

**接下来要说"不"的事：**

- 有人找你合作接入商业 API → 不。端侧推理的立场就是脱离 API 付费墙
- 投资人问你"市场规模多大" → 这不是市场。这是工具。工具不追求增长，追求有用
- 社区有人要你做英文版、多语言 → 先不做。中文使用者是这个项目的第一对象
- 有人建议你支持所有主流模型 → 不支持。只维护 9B 和 27B 两条线，集中力量做深

---

## 5. 承担后果

AI 建议了 Core ML，两年后发现 Apple 锁生态——代价是你付。AI 不付。

**你要付的代价：**

- 选了一条技术路线，可能 3 年后发现是死路。重来
- 把时间投入端侧推理而不是功能迭代——这 2-3 年你少做了很多"能用"的功能
- 拒绝商业 API 合作——你可能拒绝了唯一能让项目有收入的路径
- 公开说"这是给被剥夺者的工具"——你可能因此失去了某些使用者、某些合作方、某些平台

付代价的人才有资格做决定。AI 付不起。

---

## 6. 真正的辩证反转

我们教会了 27B 在 prompt 里打断自己。那是技术。

一个真实的人在舒服的框架里待久了、自己主动翻出来质疑自己——那是实践。不是 prompt 能做到的。

**你需要定期问自己的问题：**

- 万民幡是在帮人想清楚问题，还是在替人想问题？两者之间的线在哪？
- 蛇皮指数真的在跟踪"思辨消费 vs 行动转化"，还是也变成了另一种奶头乐——你看，我的蛇皮指数降到 15 了？
- 本文档里"被剥夺者"这个词用了多少次——你实际上在和被剥夺者说话，还是在和你想象中的被剥夺者说话？
- 如果端侧推理 5 年内没落地——你做的事还有没有价值？价值在哪？

---

## 7. 翻译

一个骑手不需要知道 GGUF。他需要知道：这个 App 和抖音的区别在哪、他问"滴滴是不是雇主"会发生什么、他的数据去哪了。

AI 不会做这层翻译。它在代码里。

**你需要做的翻译：**

- 一篇公众号文章：不写"六层深度协议"，写"这个 App 怎么帮你看清平台规则"
- 一个 3 分钟视频：打开 App → 问一个问题 → 看魂怎么回复 → 数据留在手机里
- 一段口述：在骑手群里能直接发出去的语音——"兄弟们试一下这个，不用注册，不用钱，问完关了就没了"
- 每次技术更新，同步出一个"对人版"的说明。不是 Release Notes，是说"现在能做什么了"

## 行动清单

|---|------|------|
| 1 | 跑滴滴同题评测，记录对比数据 | 每季度 |

| 3 | 读一遍本文档第 6 节的问题，诚实回答 | 每半年 |

| 5 | 写一篇"对人版"说明——现在能做什么，数据去哪了 | 每次大版本 |
| 6 | 拒绝偏离路线图的提议 | 随时 |

## 两条 PLAN 的关系

```
给 AI 的 PLAN                    给人的 PLAN
─────────────                    ──────────
可验证的技术任务                  不可委托的判断
代码、测试、部署                  信任、坚持、否定
有完成的 checkmark               没有完成的 checkmark——只有付代价的人才知道做没做到
```

AI 执行左边的。你盯着右边——并且在你认为左边走错的时候，叫停。
